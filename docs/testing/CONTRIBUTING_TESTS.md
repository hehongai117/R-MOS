# 测试贡献规范

## 适用范围
- 本规范用于任务8与任务9的测试与文档同步要求。

## 必须追加的内容
- 在 `docs/testing/TEST_PLAN.md` 追加对应任务的回归用例（每个任务至少 2 条）。
- 为新增用例分配唯一编号（示例：T8-01、T9-02）。
- 在 `DEVELOPMENT_LOG.md` 追加索引行：任务 → 提交哈希 → 用例编号范围 → ADR 编号范围。
- 若新增流程依赖种子数据，必须更新或新增脚本并在用例中引用。

## ADR 规则
- 每个任务至少 1 条 ADR 编号。
- 编号格式推荐：`ADR-TEACH-00X` 或 `ADR-OPS-00X`。

## 提交检查清单
- 用例编号与实现功能一一对应。
- 用例步骤可执行且包含状态码与关键字段断言。
- 日志索引与提交哈希匹配。
