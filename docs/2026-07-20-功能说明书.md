# R-MOS 机器人维保实训系统 · 功能说明书

> 版本：v1.0（2026-07-20）
> 产品：R-MOS（Robot Maintenance Operating System，机器人维保实训操作系统）
> 文档性质：产品功能说明书，可用于产品手册、招标功能应答、项目交付
>
> 说明：本说明书描述系统已实现的功能与业务流程。当前版本以**数字孪生仿真实训**为核心，机器人交互由仿真环境模拟，已预留真实物理机器人接入的标准扩展接口。

---

## 第一部分 · 系统概述

### 1.1 产品定位

R-MOS 是面向职业院校机器人类专业的**机器人维保实训平台**，围绕"数字孪生 + AI 智能体 + 标准化作业裁决"构建，解决维保实训中"设备贵、损耗大、演练次数受限、教师批阅工作量大、评价难量化"等痛点。

平台实现完整教学闭环：

```
教师上传机器人资料
      ↓
AI 自动分析生成教学内容（SOP 规程 / 3D 数字模型 / 故障案例库）
      ↓
学生在数字孪生环境标准化实训（3D 拆装 + 步骤操作）
      ↓
系统对每步操作自动裁决评分
      ↓
生成五维技能画像 + AI 学习反馈
```

### 1.2 核心价值

- **零真机损耗**：纯数字孪生环境，可无限次演练，不磨损真实设备。
- **一套系统多机型**：可插拔机器人平台，教师自主上传扩展新机型，非"一机一系统"。
- **AI 降本增效**：上传资料即自动生成教学内容，大幅降低教师备课成本。
- **操作自动裁决**：步骤级自动判定评分，替代人工逐一批阅。
- **能力可量化**：五维技能画像精准定位学生薄弱环节。

### 1.3 用户角色

| 角色 | 职责 | 登录后入口 |
|------|------|-----------|
| **管理员** | 系统配置、用户与权限管理、平台运维 | 管理控制台 |
| **教师** | 机器人资料管理、教学内容维护、班级监控、审批、评价 | 班级监控台 / 机器人绑定引导 |
| **学生** | 选择机器人、开展实训、查看技能画像与报告 | 学习进度面板 |

系统采用 **RBAC 基于角色的访问控制**，不同角色可见不同功能菜单，资源操作经权限校验。

## 第二部分 · 功能模块详述

### 2.1 多机器人管理平台

**功能目标**：让教师像"上传一份资料"一样把一台新机器人接入教学，无需开发人员介入。

| 功能 | 说明 |
|------|------|
| 机器人建档 | 教师创建机器人型号（品牌、型号、版本、描述），系统分配机型档案 |
| 资料上传 | 支持上传机器人 URDF 结构文件、CAD 模型、PDF 说明书等资料，批量上传 |
| AI 分析 | 上传后触发 AI 分析管线，自动解析生成 3D 装配模型、SOP 规程、故障案例 |
| 状态流转 | 机型状态：草稿 → 分析中 → 就绪，全程可视 |
| 发布闸门 | **发布前自动校验 3D 资产完整性**，缺失必需模型文件则阻断发布并明确提示缺哪个文件，从源头杜绝"上课时 3D 打不开" |
| 可见性管理 | 私有（仅本人可用）/ 共享（教师间可复用）两级 |
| 共享市场 | 优质机型资源可在教师间共享，避免重复建设 |

**内置机型**：平台内置多款主流机型资源（人形机器人、协作机械臂、灵巧手、四足机器人等），开箱即用。

### 2.2 3D 数字孪生查看器

**功能目标**：浏览器原生 3D，零客户端安装，一套渲染引擎适配所有机型。

| 功能 | 说明 |
|------|------|
| 通用渲染 | **清单（manifest）驱动**的通用 3D 引擎，任意机型只需一份清单即可渲染，无需为每台机器人定制开发 |
| 自动建模 | **URDF → GLB 自动转换管线**，教师上传机器人结构文件后自动生成可交互 3D 装配体 |
| 关节控制 | 自动生成关节控制面板，拖动滑块实时调节关节角度，模拟机器人运动 |
| 零件拆装 | 支持零件级选择、高亮、拆解、爆炸视图，还原真实拆装作业过程 |
| 视角操作 | 旋转、缩放、平移，多角度观察 |
| 降级容错 | 3D 资产缺失或渲染异常时自动降级为占位提示，不影响页面其他教学功能 |
| 技术底座 | 基于 WebGL / Three.js，浏览器直接渲染，无需 Unity / 虚幻引擎客户端或浏览器插件 |

### 2.3 标准作业规程（SOP）与智能裁决引擎 ★核心技术

**功能目标**：把"老师盯着看、事后打分"变成"系统实时裁决、自动评分"。

**SOP 管理**：

| 功能 | 说明 |
|------|------|
| 规程库 | 标准维保作业规程结构化入库，支持多机型、多难度（低/中/高）、多分类管理 |
| 步骤定义 | 每个 SOP 由有序步骤组成，含目标部件、期望动作、是否关键步骤、是否可跳过、超时时限 |
| 规程维护 | 教师可维护 SOP 内容 |

**智能裁决引擎（自主研发核心技术）**：

| 功能 | 说明 |
|------|------|
| 执行器 | 承接学生每一步操作 |
| 决策引擎 | 对操作的合规性、顺序、动作正确性实时裁决 |
| 评分器 | 逐步骤判定通过/不通过并计分 |
| 关键步骤控制 | 关键步骤必须正确完成，可跳过步骤支持灵活教学 |
| 超时控制 | 步骤超时自动处理 |
| 证据校验 | 操作证据采集与强制校验，确保规范可追溯 |

### 2.4 AI 智能教学引擎

**功能目标**：用 AI 承担教师最耗时的两件事——备课（生成内容）和批阅（生成反馈）。

| 功能 | 说明 |
|------|------|
| 内容生成 | 基于机器人资料，AI 自动生成实训项目、SOP 草稿、教学内容 |
| 反馈生成 | 学生训练提交后，AI 自动生成个性化学习反馈与改进建议 |
| 多模型路由 | 多大模型智能路由（主模型 + 备用模型 + 本地兜底），单一模型故障自动切换，保障服务可用 |
| 知识问答 | 基于知识库的检索增强问答（RAG），支持维保知识检索 |
| 多智能体协同 | 多智能体协同编排，支持复杂诊断/执行任务的智能体协作 |
| 超时保护 | AI 调用全链路超时保护，避免长时间无响应 |

> 说明：AI 大模型可采用云端 API 或本地私有化部署两种形态，按数据合规要求选择（本地私有化需配置 GPU 算力）。

### 2.5 训练与实操流程

**功能目标**：为学生提供从选机器人到看报告的完整实训动线。

| 功能 | 说明 |
|------|------|
| 机器人上下文 | 学生登录后选择当前实训机器人，系统记住上下文；仅有一台时自动选中 |
| 场景选择 | 自主练习场景库，学生按需选择实训场景 |
| 实操工作台 | SOP 实操工作台：**3D 视图 + 步骤面板 + 裁决反馈**一体化界面，边看 3D 边执行步骤边得到判定 |
| 会话管理 | 完整训练会话状态机：进行中 → 暂停 → 继续；支持中断续训 |
| 会话结束 | 支持手动提交、超时提交、教师提交；会话超时（48 小时）自动处理；支持放弃 |
| 进度跟踪 | 实时跟踪训练进度 |

**学生黄金路径**：

```
登录 → 学习进度面板（选机器人）→ 场景选择 → SOP 实操工作台
      →（逐步操作 + 实时裁决）→ 提交 → 维保报告 / 技能画像
```

### 2.6 考核评价与技能画像

**功能目标**：把"分数"升级为"能力雷达图"，让教学更有针对性。

**五维技能画像**：系统从以下五个维度自动评估并生成能力画像——

| 维度 | 评估内容 |
|------|---------|
| **安全** | 安全规范执行情况 |
| **步骤** | 操作步骤的规范性 |
| **精度** | 操作精度 |
| **效率** | 时间效率 |
| **工具** | 工具使用规范性 |

| 功能 | 说明 |
|------|------|
| 自动评分 | 多维度自动评分，教师无需逐一人工批阅 |
| 技能画像 | 生成学生个人五维技能雷达图，定位薄弱维度 |
| 学习记忆 | 训练记忆持续写入，跟踪学生成长轨迹 |
| 维保报告 | 自动生成实训报告，可查询回溯 |

## 第三部分 · 典型业务流程

### 3.1 教师侧：接入一台新机器人并投入教学

1. **建档**：教师登录，进入机器人管理，创建机器人型号（填写品牌、型号等）。
2. **上传资料**：上传该机器人的 URDF 结构文件、CAD 模型、PDF 说明书。
3. **AI 分析**：触发分析管线，系统自动生成 3D 装配模型、SOP 规程草稿、故障案例。
4. **审核完善**：教师审核 AI 生成内容，按需调整 SOP。
5. **发布**：点击发布，系统自动校验 3D 资产完整性——齐全则发布为"就绪"，缺失则阻断并提示。
6. **共享（可选）**：将机型设为共享，供其他教师复用。
7. **投入教学**：学生即可在该机器人上开展实训。

### 3.2 学生侧：完成一次标准维保实训

1. **登录**：学生登录，进入学习进度面板。
2. **选机器人**：从可用机器人中选择本次实训对象（仅一台时自动选中）。
3. **选场景**：进入场景选择页，挑选实训场景。
4. **进入工作台**：跳转 SOP 实操工作台，左侧/主区 3D 视图，侧边步骤面板。
5. **逐步操作**：按 SOP 步骤在 3D 模型上执行拆装/检查/操作，系统对每步实时裁决并反馈通过/不通过。
6. **提交**：完成后提交，或超时/教师提交。
7. **看结果**：查看 AI 学习反馈、五维技能画像、维保报告。

### 3.3 教师侧：监控与评价

1. 教师在班级监控台实时查看学生实训状态。
2. 通过审批队列处理需审批的关键操作。
3. 查看班级/个人技能画像，识别共性薄弱环节，调整教学重点。

## 第四部分 · 系统特性

### 4.1 可靠性

- **1200+ 自动化测试**（后端 791 项、前端 465 项）持续保障功能正确性。
- **浏览器级端到端测试**覆盖登录→实训→报告黄金路径。
- **错误容错**：路由级 / 3D 渲染级错误边界，单点异常不导致整页崩溃。
- **网络容错**：API 瞬时失败自动重试，弱网体验保障。

### 4.2 可部署性

- **零客户端**：师生浏览器直接使用。
- **一键部署**：Docker 容器化，支持校内私有化部署或云端部署。
- **存储弹性**：本地存储 / 对象存储（S3/OSS/MinIO）平滑切换，满足扩容。
- **平滑升级**：数据库版本化迁移，升级不丢数据。

### 4.3 开放性

- 存储后端、AI 大模型、部署方式均可切换，不锁定单一供应商。
- 预留真实物理机器人接入的标准适配器接口，支持后续软硬件一体化扩展。
