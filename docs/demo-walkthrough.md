# R-MOS MVP Demo 操作手册

> 投资人演示流程，完整走一遍约 5-8 分钟。
> 演示故事线：**发现故障 → AI 诊断 → 3D 引导维保 → 生成报告**

---

## 0. 启动准备

### 0.1 启动后端

```bash
cd r-mos-backend
source .venv/bin/activate      # 或 source venv/bin/activate，取决于你的虚拟环境名
python scripts/seed_demo_data.py   # 首次运行需注入演示 SOP 数据（已有则跳过）
python main.py                     # 默认 http://localhost:8000
```

### 0.2 启动前端（Demo 模式）

```bash
cd r-mos-frontend
VITE_DEMO_MODE=true npm run dev    # 默认 http://localhost:3000
```

> **关键：** 必须加 `VITE_DEMO_MODE=true`，否则不会进入演示模式。

### 0.3 打开浏览器

访问 `http://localhost:3000`，使用以下账号登录：

- 用户名：`admin@rmos.test`
- 密码：`Admin@123`

### 0.4 确认 Demo 模式生效

登录后检查两个标志：

1. **左下角用户头像旁**有蓝色 `Demo` 徽章
2. **侧边栏只显示 4 个页面**：
   - 实时监控
   - AI 诊断工作台
   - 维保工作台
   - 维保报告

如果侧边栏显示了更多页面（如"训练中心"、"知识库"等），说明 `VITE_DEMO_MODE` 未生效，请重新启动前端。

## Act 1：故障发现（实时监控页）

### 页面位置

侧边栏点击 **「实时监控」**，或直接访问 `http://localhost:3000/monitor`

### 操作步骤

1. **触发故障模拟**
   - 页面右上角有一个蓝色按钮 **「触发故障演示」**
   - **点击该按钮**，页面中部会出现蓝色提示条：
     > 故障模拟已触发 — 等待左膝温度升高后，**点击右侧变红的「左膝关节」卡片**进入 AI 诊断

2. **观察温度变化**（约 30 秒）
   - 右侧「重点关节」区域中，左膝关节（knee_left）温度从 ~36°C 开始逐渐上升
   - **~55°C 时**：左膝关节卡片边框变为**琥珀色/黄色**闪烁（温度预警）
   - **~65°C 时**：左膝关节卡片变为**红色**，显示错误状态

3. **点击告警卡片跳转**
   - 当左膝关节卡片变红后（或琥珀色预警时），**单击该关节卡片**
   - 页面会自动跳转到 **AI 诊断工作台**，URL 携带参数：
     `?fault=knee_overheat&joint=knee_left`

> **演示话术提示：** "系统实时监控机器人各关节传感器数据，当检测到左膝关节温度异常升高时，自动触发告警。运维人员点击告警即可进入 AI 辅助诊断。"

---

## Act 2：AI 诊断（Agent 工作台页）

### 页面位置

从 Act 1 自动跳转而来，或手动访问：
`http://localhost:3000/agent/workbench?fault=knee_overheat&joint=knee_left`

### 操作步骤

1. **确认自动填充的告警信息**
   - 页面加载后，对话流中会自动出现一条**智能体消息**：
     > 检测到设备告警：**knee_left** 温度异常升高至 65°C，已超过安全阈值。
     > 请输入"诊断"开始故障分析，或直接描述您的需求。

2. **发送诊断指令**
   - 在底部输入框中输入 **「诊断」**
   - 点击 **「发送」** 按钮（或按 Enter 键）

3. **观察 SSE 流式响应**（约 3-5 秒）
   - 智能体会以**打字机效果**逐字输出故障诊断报告
   - 内容包括：故障类型、可能原因、影响部件、建议措施等

4. **查看右侧诊断面板**
   - 页面右侧的 **「诊断结果」** 面板会同步更新：
     - **H1 · BEARING_WEAR** — 左膝关节轴承磨损
     - 置信度：**92%**
     - 受影响部件：`left_knee_bearing`, `left_knee_joint`
     - 证据数据：温度/扭矩/电流异常
     - 可能原因：轴承长期运转磨损、润滑不足加速磨损

5. **点击"开始维保"**
   - 流式响应完成后，对话区域底部出现蓝色按钮：
     **「开始维保 → ATOM-01 左膝关节轴承更换」**
   - 点击该按钮，自动跳转到 **维保工作台**

> **演示话术提示：** "AI 智能体基于多维传感器数据进行故障推理，实时给出诊断结论和置信度，并推荐对应的标准维保流程。"

## Act 3：3D 引导维保（维保工作台页）

### 页面位置

从 Act 2 自动跳转而来，或手动访问：
`http://localhost:3000/maintenance?sop=knee-bearing-replace`

### 操作步骤

1. **确认 SOP 自动加载**
   - 左侧面板显示 **「ATOM-01 左膝关节轴承更换」** SOP
   - 列出 6 个步骤：安全确认 → 工具准备 → 外壳拆卸 → 轴承定位 → 轴承更换 → 回装验证
   - 右侧 3D 视图自动聚焦到**左膝关节**

2. **逐步执行 6 个维保步骤**

   对每个步骤重复以下操作：

   **a)** 点击底部播放器区域的 **「下一步」** 按钮 → 步骤进入「执行中」状态
   
   **b)** 步骤会显示提示："等待实际操作完成，请点击手动验证继续"
   
   **c)** 点击 **「手动验证」** 按钮 → 步骤标记为已完成 ✓
   
   **d)** 自动进入下一步

   6 个步骤概览：

   | 步骤 | 名称 | 说明 |
   |------|------|------|
   | 01 | 安全确认 | 断电确认维保隔离 |
   | 02 | 工具准备 | 确认扳手、拔取器、润滑脂 |
   | 03 | 外壳拆卸 | 拆卸 4 颗 M3 螺丝 |
   | 04 | 轴承定位 | 定位磨损轴承并记录 |
   | 05 | 轴承更换 | 拔取旧轴承、安装新轴承 |
   | 06 | 回装验证 | 回装外壳、通电测试 |

3. **完成后自动跳转**
   - 第 6 步完成后，页面会显示"已完成"状态
   - **约 2 秒后自动跳转**到维保报告页

> **演示话术提示：** "3D 引导系统将复杂的维保操作分解为标准步骤，每一步都有明确指引和验证，确保操作规范性。"

## Act 4：维保报告（报告页）

### 页面位置

从 Act 3 自动跳转而来（URL：`/reports/demo`）

### 查看内容

报告页面自动展示以下内容（无需额外操作）：

1. **故障诊断摘要**（顶部卡片）
   - 故障类型：左膝关节轴承磨损
   - AI 置信度：92%
   - 风险评级：中高
   - 处置结果：维保完成

2. **维保前后对比**
   - 维保前温度：**65°C**（红色）
   - 维保后温度：**35°C**（绿色）

3. **诊断证据链**
   - 🌡️ 左膝温度 35→65°C（30s 内）
   - ⚙️ 左膝扭矩波动 ±2.1Nm
   - ⚡ 左膝电流 2.0→2.8A
   - 📋 上次维保距今 180 天，超出建议周期

4. **评分统计**
   - 最终得分：**92 分**
   - 总时长、完成步骤（6/6）、错误次数（0）
   - 五维评分分解：Safety / Procedure / Precision / Efficiency

5. **步骤得分详情**（表格）
   - 每一步的得分、扣分项、备注

> **演示话术提示：** "系统自动生成标准化维保报告，包含故障诊断证据、维保效果对比和操作评分，实现维保质量的可追溯和可量化。"

## 快速重置（重新演示）

如果需要重新演示一遍：

1. 刷新浏览器页面
2. 回到 **实时监控** 页面（`/monitor`）
3. 再次**双击标题**触发故障
4. 从 Act 1 重新开始

> 离开监控页时，故障状态会自动重置。

## 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 双击标题无反应 | 不在 Demo 模式 | 检查前端是否以 `VITE_DEMO_MODE=true` 启动 |
| 点击关节卡片无反应 | 温度未达到告警阈值 | 等待温度升至 55°C 以上（琥珀色预警） |
| 诊断流式文字不出现 | 后端未启动或端口不对 | 检查后端 8000 端口是否正常运行 |
| 维保页面 SOP 未自动选择 | URL 缺少 sop 参数 | 从 Agent 页点击"开始维保"按钮跳转 |
| 完成 6 步后未跳转报告 | 步骤未全部完成 | 确认所有 6 步都显示 ✓ |
| 侧边栏显示过多页面 | Demo 模式未生效 | 重启前端：`VITE_DEMO_MODE=true npm run dev` |
