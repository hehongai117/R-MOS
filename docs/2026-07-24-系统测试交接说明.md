## 二、如何把系统跑起来

```bash
# 后端（需本机 PostgreSQL 在跑，rmos 库已迁移+种子）
cd r-mos-backend && source venv/bin/activate && python main.py     # → :8000

# 前端（dev 代理 /api → :8000）
cd r-mos-frontend && npm run dev                                    # → :3000
```

**演示账号**（种子数据 `scripts/seed_demo_full.py` 产出，密码可改）：
- 管理员：`admin@rmos.test` / `Admin@123`
- 教师：`teacher1@rmos.demo` / `Teacher@123`（有完整班级/学生/机器人数据）
- 学生：`student1@rmos.demo` / `Student@123`

> 若本机库数据被测坏，`python scripts/seed_demo_full.py --reset` 可清并重建演示数据（幂等）。

## 三、⚠️ 已知设计限制 / 非缺陷（避免误报为 bug）

系统测试前请先看这一节——以下都是**当前阶段的有意设计或已知项，不是 bug**：

1. **机器人交互全部由 Mock 适配器模拟**（MVP 阶段）。系统没有连真实物理机器人；遥测数据是模拟生成的。
2. **本地无真实大模型 API key 时，AI 走 mock 兜底**。AI 生成的项目/SOP/反馈在本地是桩数据；要测真实 AI 输出需配置 DeepSeek/MiniMax key（`.env`）。
3. **只有 3 台机器人 3D 资产完整**：ATOM-01、GRx-N1、6DOF-A-06。其余种子机器人因资产不全已被资产闸门置为草稿（学生只能在这 3 台上实训，是正确行为）。
4. **integration-ci.yml 在 push 时会显示一条 0-job 的 "failure"**：这是 GitHub 对"只在 PR 触发的 workflow"在 push 改动其文件时的空记录，**不是真失败**（无 job、无日志）。真实验证在 PR 时触发。
5. **dev 构建工具链有 12 个依赖漏洞**（rollup/picomatch 等），需破坏性升级，已暂缓；**生产依赖 0 漏洞**。
6. **Pydantic V2 class-based Config 弃用告警**：cosmetic，不影响功能。
7. **未部署到云端**：无 staging/prod 环境，容器化/CI 已就绪但 CD 未做（属路线图 P1-3）。
8. **性能**：SOP 3D 工作台首屏已优化（90MB→13MB）；真实 LLM 延迟、3D 交互帧率本地测不了（需带真 key 的 staging，见 `phase4-baseline.md`）。

## 四、建议系统测试重点覆盖的功能域

自动化测试已覆盖单元/接口/黄金路径；系统测试建议重点做**跨角色、跨模块、边界与探索性**验证：

| 功能域 | 测试关注点 |
|--------|-----------|
| 教师全流程 | 建机器人→上传资料→AI 分析→审核→发布（资产闸门）→共享；班级监控；审批队列 |
| 学生全流程 | 选机器人→选场景→SOP 实操（3D 拆装+步骤裁决）→提交→报告→技能画像 |
| 多机器人切换 | 在 3 台完整机型间切换上下文，SOP/3D/故障库是否正确随机型变化 |
| SOP 裁决引擎 | 遍历 SOP 库，正确/错误/跳步/超时操作的裁决是否合理（核心 IP，重点测） |
| 五维技能画像 | 安全/步骤/精度/效率/工具 的评分是否与操作表现一致 |
| 训练会话状态机 | 进行中→暂停→继续；超时提交；教师提交；放弃；48h 超时 |
| RBAC 权限边界 | 学生越权访问教师/管理端；跨用户访问他人资源；机器人 owner 校验 |
| 文件上传/分析管线 | URDF/CAD/PDF 上传→AI 分析→3D/SOP/故障库生成的鲁棒性（异常文件、超大文件） |
| 证据系统 | 操作证据采集与强制校验 |
| WebSocket 遥测 | 5Hz 实时数据、断线重连、弱网表现 |
| 兼容性 | 不同浏览器（Chrome/Edge）、不同分辨率、3D WebGL 表现 |

## 五、可交回工程团队的技术债清单（测试中若遇到，对照此表）

详见 `docs/项目交接与升级路线图.md` 第三部分技术债总表与各梯队。摘要：

- **待做（需外部决策才能开工）**：P1-3 云部署+域名（需定部署目标/镜像仓库）、P1-4 Sentry 错误告警（需 DSN）、P3 战略项（真机接入 vs 纯孪生、多租户 SaaS）。
- **已登记的 Defer 项**：3 个遗留 timestamp 列 naive（conversation_turns/sop_audit_logs，与迁移链一致无漂移）；dev 依赖漏洞；Pydantic V2 弃用清理；protected-vitals.mjs 补软件 WebGL 开关；可选 GLB Draco 压缩（3D 从 13MB 再降）。
- **逆向建模路线（业务决定）**：非合作机型走"逆向 CAD→导 URDF→上传"，系统现有管线已支持（URDF+网格→自动结构图/爆炸图），无需新功能。建议正式逆向前用一个公开标准 URDF 验证管线端到端通畅。
