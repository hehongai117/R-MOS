### 1.2 核心故事线
**左膝关节过热 → AI 诊断为轴承磨损 → 3D 引导维保 → 验收通过**

- **原则**：不删除任何现有代码，只增不破坏

## 2. 演示脚本（10-15 分钟）

### 开场（1 分钟）
- 登录系统，简述 R-MOS 是什么

### Act 1 - 发现故障（3 分钟）
- 进入「实时监控」，3D 机器人实时展示关节状态
- 演示者触发故障模拟（双击 logo 隐藏触发）
- 左膝关节温度从 35°C 逐渐升高到 65°C（30 秒内），触发告警
- 3D 模型上左膝关节高亮变红
- 右侧重点关节面板显示异常数值（温度、扭矩、电流）

### Act 2 - AI 诊断（4 分钟）
- 点击告警，跳转「AI 诊断工作台」
- 系统自动带入故障上下文："左膝关节温度异常，当前 65°C"
- 用户输入："帮我诊断这个故障"
- AI（mock）以打字机效果返回结构化诊断结果：
  - 故障判定：左膝关节轴承磨损导致摩擦过热
  - 证据引用：温度趋势、扭矩异常、历史数据对比
  - 风险评级：中高（需要尽快维保）
  - 推荐方案：执行 SOP "左膝关节轴承更换"
- 用户点击"开始维保"

### Act 3 - 3D 引导维保（4 分钟）
- 跳转「维保工作台」，左侧步骤列表，右侧 3D 模型
- 逐步执行 6 个步骤：
  1. 安全确认 - 断电并确认维保隔离
  2. 工具准备 - 确认扳手、轴承拔取器、润滑剂就位
  3. 外壳拆卸 - 拆卸左膝关节保护外壳（3D 爆炸视图展开）
  4. 轴承定位 - 定位磨损轴承（3D 高亮具体部件）
  5. 轴承更换 - 拔取旧轴承，安装新轴承
  6. 回装验证 - 回装外壳，通电，关节活动度测试
- 每一步 3D 视角自动切换，高亮对应部件

### Act 4 - 维保验收（2 分钟）
- 维保完成，自动生成维保报告
- 展示：故障摘要、AI 诊断结论、维保过程时间线、证据包
- 展示维保前后对比：温度从 65°C 恢复到 35°C
- 报告底部展示 AI 诊断引用链

### 收尾（1 分钟）
- 总结产品价值，展望教学/知识库等扩展能力

## 3. 页面设计

### 3.1 保留页面（4 个）

#### 页面 1：实时监控（MonitorPage）

**现状**：WebSocket 连接正常，3D 模型联动，10 路关节数据实时显示。

**改动**：
- 修复电池数值浮点精度问题（`59.93999999999774` → `59.9%`）
- 新增故障告警：关节温度超过 55°C 时，该关节卡片变为红色告警状态
- 3D 模型上对应关节高亮变红
- 新增隐藏故障触发按钮（双击 logo）
- 告警卡片可点击，跳转到 Agent 工作台并携带故障上下文参数

#### 页面 2：AI 诊断工作台（AgentWorkbenchPage）

**现状**：有 UI 壳子，对话流、状态卡片、诊断结果面板都在，但没有 AI 响应。

**改动**：
- 接入 mock LLM 响应服务，根据意图关键词返回预设结构化回答
- 诊断结果面板接入 mock 数据：故障类型、证据引用、风险评级、推荐 SOP
- 新增"开始维保"按钮，跳转到 SOP 工作台
- 响应通过 SSE 流式推送，模拟打字机效果

#### 页面 3：维保工作台（SOPMaintenancePage）

**现状**：有步骤列表和 3D 模型，有爆炸视图功能。步骤内容为"躯干外观与连接点检"。

**改动**：
- 步骤内容替换为"左膝关节轴承更换" SOP（6 步）
- 每个步骤绑定 3D 视角切换（扩展 Left Knee Service 视角）
- 步骤执行时 3D 爆炸视图联动
- 每步完成后记录时间戳

#### 页面 4：维保报告（ReportPage）

**现状**：页面存在。

**改动**：
- 展示完整维保报告：故障摘要、AI 诊断结论、SOP 执行步骤+时间线
- 展示维保前后对比（温度 65°C → 35°C）
- 报告底部展示 AI 诊断引用链（证据溯源）

### 3.2 导航精简

## 4. 技术方案

### 4.1 Mock LLM 服务

新增 `app/services/llm/mock_provider.py`

**意图匹配规则**：

| 用户输入包含 | 返回预设 |
|---|---|
| "诊断" / "什么问题" / "故障" | 结构化诊断报告（轴承磨损、证据引用、风险评级） |
| "维保方案" / "怎么修" / "SOP" | 维保 SOP 步骤列表（6 步） |
| "解释" / "为什么" | 故障原因深度解释（温度-扭矩关联分析） |
| 其他 | 通用回复引导用户提问 |

**响应格式**：
```json
{
  "text": "markdown 格式的自然语言回复",
  "diagnosis": {
    "fault_type": "bearing_wear",
    "joint": "KNEE_LEFT",
    "severity": "high",
    "confidence": 0.92,
    "evidence": ["温度趋势异常", "扭矩波动增大", "历史维保记录"]
  },
  "citations": [
    {"type": "sensor", "desc": "左膝温度 35→65°C（30s）"},
    {"type": "sensor", "desc": "左膝扭矩波动 ±2.1Nm"},
    {"type": "history", "desc": "上次维保距今 180 天"}
  ],
  "sop_draft": null
}
```

**流式输出**：通过 SSE（Server-Sent Events）逐字推送 text 字段，模拟 LLM 打字机效果。

### 4.2 故障模拟

扩展现有 `MockRobotAdapter`：

**新增 API**：`POST /api/v1/adapter/simulate-fault`
```json
{
  "fault_type": "knee_overheat",
  "joint": "KNEE_LEFT"
}
```

**效果**：mock adapter 在 30 秒内将左膝温度从 35°C 线性升高到 65°C，同时扭矩和电流数值相应异常化。数据通过现有 WebSocket 管线实时推送。

**重置 API**：`POST /api/v1/adapter/reset-fault` — 温度在 10 秒内恢复正常。

**触发方式**：监控页双击 R-MOS logo 触发，演示者无痕操作。

### 4.3 页面跳转串联

```
/monitor（发现告警）
  → 点击告警卡片
  → /agent/workbench?fault=knee_overheat&joint=KNEE_LEFT

/agent/workbench（AI 诊断）
  → 对话后点击"开始维保"
  → /maintenance?sop=knee_bearing_replace

/maintenance（执行 SOP）
  → 完成所有步骤后
  → /reports/{task_id}

/reports/{task_id}（维保验收）
  → 展示完整报告
```

### 4.4 Seed 数据

新建 `scripts/seed_demo_data.py`：

**SOP 数据**：
- 名称：ATOM-01 左膝关节轴承更换
- 分类：关节维保
- 难度：intermediate
- 预估时长：45 分钟
- 6 个步骤（安全确认 → 工具准备 → 外壳拆卸 → 轴承定位 → 轴承更换 → 回装验证）

**历史数据**：
- 一条历史维保记录（让报告页"历史对比"有数据）
