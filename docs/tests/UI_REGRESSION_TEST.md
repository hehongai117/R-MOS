# UI 回归测试清单（裁决级重构 UAT）

> 目标：作为裁决级系统重构的最终人工验收标准，覆盖教学 / 考试 / 维保模式的关键闭环与视觉反馈。

## 场景 A：教学模式下的引导闭环

**前置条件**
1. 进入 `SOP 维保系统` 页面。
2. 模式切换选择为「教学模式」。
3. 选中任意 SOP 脚本（例如：躯干或脚部拆卸流程）。

**步骤 / 操作 / 预期视觉反馈**
1. **Step**：执行一个带前置条件的步骤，但故意不满足前置条件（例如：未先拆软胶直接分离脚底板）。  
   **Action**：点击「下一步」或执行按钮。  
   **Expected Visual Result**：出现**红色错误提示**（“操作被阻断”），其**正下方**出现一个**蓝色背景**的“教学提示气泡”（带**灯泡图标**）。

2. **Step**：观察教学提示气泡内容。  
   **Action**：阅读提示内容（来自 SOP 的 teachingResponse.hintContent）。  
   **Expected Visual Result**：气泡中显示明确指导文案，例如“请先移除软胶，再继续拆卸”。

3. **Step**：点击气泡右侧的「重试」按钮。  
   **Action**：点击“重试 (Retry)”按钮。  
   **Expected Visual Result**：红色错误提示与蓝色气泡**同时消失**，步骤状态回到可执行状态（IDLE）。

## 场景 B：考试模式下的普通扣分

**前置条件**
1. 切换模式为「考试模式」。
2. 顶部出现考试栏（倒计时 + 当前得分）。

**步骤 / 操作 / 预期视觉反馈**
1. **Step**：触发一个普通错误（但允许继续），如未满足前置条件。  
   **Action**：点击「下一步」。  
   **Expected Visual Result**：出现**红色错误提示**（仅红字），**不出现蓝色教学提示气泡**。

2. **Step**：观察顶部考试栏分数变化。  
   **Action**：等待 0.5 秒。  
   **Expected Visual Result**：得分从 **100 → 90**，数字出现**放大/闪烁**的视觉反馈（短暂缩放或高亮）。

3. **Step**：观察倒计时。  
   **Action**：静待 5 秒。  
   **Expected Visual Result**：倒计时正常递减（mm:ss），在剩余 <5 分钟时变红。

## 场景 C：考试模式下的致命熔断

**前置条件**
1. 保持「考试模式」。
2. 选择包含 `fatalOnFailure` 或 `allowContinue=false` 的步骤。

**步骤 / 操作 / 预期视觉反馈**
1. **Step**：触发严重错误（不可逆/严重约束）。  
   **Action**：执行错误操作或点击「下一步」。  
   **Expected Visual Result**：系统**自动弹出全屏遮罩**“考试结束”。

2. **Step**：检查遮罩信息。  
   **Action**：查看遮罩上的信息。  
   **Expected Visual Result**：遮罩上**大字显示** `Reason Code` 与**最终分数**（例如“原因码：ERR_FATAL_*”“最终得分：0/90”）。

3. **Step**：尝试与背景交互。  
   **Action**：尝试点击 SOP 面板或 3D 视图。  
   **Expected Visual Result**：**背景被锁定**，无法继续任何操作。

## 场景 D：模式切换的数据隔离

**前置条件**
1. 进入「考试模式」并触发一次扣分（分数非 100）。

**步骤 / 操作 / 预期视觉反馈**
1. **Step**：切换至「教学模式」。  
   **Action**：点击右上角模式选择下拉框，选择「教学模式」。  
   **Expected Visual Result**：弹出二次确认框，提示“切换模式将重置当前进度，确定吗？”。

2. **Step**：确认切换。  
   **Action**：点击“确定”。  
   **Expected Visual Result**：  
   - 分数恢复为 **100**（考试栏消失）；  
   - SOP 步骤进度重置；  
   - UI 恢复初始状态（错误提示、考试遮罩全部消失）。
