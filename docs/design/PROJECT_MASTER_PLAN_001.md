1. 严格遵循 `AGENTS.md` 与 `docs/testing/ACCEPTANCE_CHARTER.md`

## 2.4 任务4：完整自动化回归（Batch-1）

### 目标

执行当前基线第一轮全集回归并锁定问题清单。

### 子任务

1. 后端 unit：`pytest -q tests/unit -q`
2. Gate2 smoke：`./scripts/run_gate2_smoke.sh`

### 完成判定

1. 每条命令有输出摘要和退出码
2. 失败项有错误证据与归因
3. 形成批次编号（Batch-1）与缺陷池

## 2.5 任务5：新功能逐项“可用/不可用”核证

### 目标

对全部新增功能给出二元结论并可追溯。

### 子任务

1. 按功能维度执行 happy path
2. 按权限/对象级执行负路径（404/403 语义）
3. 校验审计追踪（trace_id 贯通）
4. 形成“可用/不可用”判定表

### 可用判定标准（必须同时满足）

1. 主路径 PASS
2. 安全语义 PASS（鉴权/审批/对象级）
3. 审计可追溯 PASS
4. 对应 Test ID 证据完整

### 不可用判定标准

1. 主路径失败
2. 负路径语义错误（应 404/403 却异常）
3. 无审计/trace 断链
4. 无可复现入口或证据缺失

---

## 2.6 任务6：缺陷分级、修复与记录

### 目标

以最小批次清零阻断交付的缺陷。

### 子任务

1. 缺陷分级：P0（阻断交付）、P1（高风险）、P2（可延期）
2. 修复节奏：先 P0，再 P1，最后评估 P2
3. 一缺陷一提交，禁止混修
4. 每次修复必须跑相关最小回归

### 完成判定

1. P0 全清零
2. P1 具备可接受风险或已清零
3. 每个缺陷有“根因 + 修复提交 + 回归证据”
