## 2. 开发策略

### 2.1 核心原则

```
1. 保留层  →  复用现有视觉组件和 3D 渲染
2. 新增层  →  创建裁决引擎、状态机、约束图
3. 改造层  →  将现有组件接入新增层
4. 验证层  →  用脚部总成进行垂直切片验证
```

#### 2.2 裁决引擎核心（1 天）

**文件**: `src/adjudication/core/decisionEngine.ts`

| 函数 | 说明 | 返回值 |
|------|------|--------|
| `canOperatePart(partId, action, state)` | 判断是否可以操作零件 | `AdjudicationReport` |
| `canRemoveScrew(screwId, toolId, state)` | 判断是否可以拆螺丝 | `AdjudicationReport` |
| `adjudicateAction(action, state)` | 裁决操作结果 | `AdjudicationReport` |
| `getBlockingConstraints(partId, state)` | 获取阻止操作的约束 | `Constraint[]` |

**裁决结果类型**:
```typescript
enum AdjudicationResult {
  ALLOWED = 'allowed',
  BLOCKED = 'blocked',
  TOOL_MISMATCH = 'tool_mismatch',
  INCOMPLETE = 'incomplete',
  WARNING = 'warning'
}
```

**验收标准**:
- 拆卸顺序错误时返回 `BLOCKED` + 原因
- 工具不匹配时返回 `TOOL_MISMATCH` + 正确工具
- 螺丝未完全拆除时返回 `INCOMPLETE` + 剩余量

```typescript
interface AdjudicationState {
  // 系统状态
  systemState: SystemState;  // 'fully_assembled' | 'partial_disassembly' | ...

  // 螺丝状态
  screwStates: Record<string, ScrewState>;  // { state, rotations, zDisplacement }

  // 当前工具
  currentToolId: string | null;
  
  // 操作历史
  actionHistory: ActionRecord[];
  
  // 状态变更函数
  setScrewState: (screwId: string, state: ScrewState) => void;
  setPartRemoved: (partId: string) => void;
  setCurrentTool: (toolId: string) => void;
  resetState: () => void;
}
```

**验收标准**:
- 状态变更可追踪
- 能通过 `getState()` 获取完整系统状态
- 能重置到初始状态

| 函数 | 说明 |

#### 3.2 SOP 脚本格式升级（0.5 天）

**文件**: `src/data/sopScripts.ts` (改造)

**升级前**:
```typescript
interface SOPStep {
  id: number;
  type: SOPStepType;
  title: string;
  description: string;
  partName?: string;
  screwId?: string;
  toolId?: string;
}
```

**升级后**:
```typescript
interface SOPStep {
  stepId: string;
  stepIndex: number;
  title: string;
  description: string;
  action: ActionType;
  targetParts: string[];
  requiredTool: string | null;
  
  // 新增：前置条件
  preconditions: SOPPrecondition[];
  
  // 新增：完成验证
  validations: SOPValidation[];
  
  // 新增：错误裁决
  failureReasons: SOPFailureReason[];
  
  // 新增：状态转移
  onSuccess: { nextStepId: string; stateTransition: SystemState | null };
  onFailure: { action: 'block' | 'warn' | 'retry'; message: string };
}
```

**验收标准**:
- 现有 SOP 脚本升级为新格式
- backward compatible（兼容旧格式）

## 7. 开发规范

### 7.1 代码规范

```typescript
// 所有裁决函数必须返回 AdjudicationReport
function canXxx(...): AdjudicationReport { ... }

// 所有状态变更必须通过 stateManager
stateManager.setScrewState(id, state);  // ✅
screwStates[id] = state;                 // ❌

// 动画完成必须回调裁决引擎
onAnimationComplete={() => {
  const report = decisionEngine.adjudicate(...);
  if (report.result !== 'allowed') {
    rollbackAnimation();
    showError(report.reason);
  }
}}
```

### 7.2 测试规范

每个模块必须有单元测试：
- `decisionEngine.test.ts`
- `stateManager.test.ts`
- `sopExecutor.test.ts`

每个新文件必须有 JSDoc 注释：
```typescript

 * @description 裁决引擎核心模块
 * @module adjudication/core/decisionEngine
```
