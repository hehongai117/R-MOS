**一、改造目标与边界定义**

**1.1 改造目标**

将 R-MOS 从**「规则驱动的智能体治理骨架」**升级为**「大模型驱动的原生教学智能体（LLM-Native Agent）」**，使系统能够以自然语言理解用户意图、以 LLM 推理替代规则判断、以统一记忆/知识中台支撑跨会话学习。

|                         |              |              |          |                                                                       |
|-------------------------|--------------|--------------|----------|-----------------------------------------------------------------------|
| **评估维度（权重）**    | **当前得分** | **目标得分** | **差距** | **改造核心动作**                                                      |
| **LLM 推理核心（30%）** | **0 / 10**   | **8 / 10**   | **严重** | 接入真实 LLM Provider，构建意图理解/SOP裁决/教学对话/故障诊断四大引擎 |
| **架构完整性（20%）**   | **5 / 10**   | **9 / 10**   | **较高** | 收敛双轨 Agent（/ai/\* + /agent/v2/\*）为单一主链路                   |
| **记忆与知识（20%）**   | **4 / 10**   | **8 / 10**   | **较高** | 建立统一三层记忆中台，打通 RAG 知识双轨                               |
| **安全治理（15%）**     | **8 / 10**   | **9 / 10**   | **良好** | 保留审批/审计链，增加 LLM 动态风险评分层                              |
| **前端真实度（10%）**   | **5 / 10**   | **9 / 10**   | **中等** | 消除 ReplayPage/ApprovalQueue/AcceptanceDash 等关键 Mock              |
| **教学效果（5%）**      | **6 / 10**   | **9 / 10**   | **中等** | 前置检查、失败分级、个性化报告、群体对比四项教学增强                  |

**5.1 LLM 调用规范**

**5.2 工具调用规范**

- 所有工具必须在 ToolRegistry 中注册，包含：name / description / parameters JSON Schema / risk_level / requires_approval

- risk_level=HIGH 的工具（side_effects=true）调用前必须经过 ApprovalService，对接现有 approvals 表

- 工具执行结果必须写入 ai_tool_calls 表：input / output / duration_ms / error / status

- 工具超时统一 30s，超时后 Agent 自动切换降级策略，不挂起等待

**5.3 提示模板规范**
